# Code Agent Traffic Light 架构文档

本文记录第一阶段已经落地的架构、模块边界和运行机制。

## 1. 项目目标

Code Agent Traffic Light 是一个跨平台桌面透明悬浮状态灯，用于显示本机 Codex 和 Claude 的工作状态。

第一阶段目标：

- 提供 macOS / Windows 桌面悬浮窗
- 动态检测运行中的 `codex` / `claude` CLI 进程
- 通过 hooks 自动更新状态灯
- 通过 CLI 手动控制状态
- 使用本地状态文件持久化
- 支持开发模式和 DMG 打包安装

第一阶段不做：

- 自动修改用户的 `~/.codex/config.toml`
- 自动修改用户的 `~/.claude/settings.json`
- 多会话精细区分
- 云同步
- 移动端
- 硬件红绿灯

## 2. 技术栈

- 桌面端：Electron
- 前端：React + TypeScript
- 构建：Vite
- 打包：electron-builder
- CLI / hook 脚本：Node.js
- 测试：Vitest

Electron 的职责是提供透明悬浮窗、置顶能力、托盘菜单、跨平台窗口控制和本地进程轮询。

## 3. 核心状态模型

支持的 agent：

```ts
type AgentId = "codex" | "claude";
```

支持的状态：

```ts
type AgentState =
  | "idle"
  | "working"
  | "done"
  | "waiting"
  | "error";
```

状态含义：

| 状态 | 灯色 | 含义 |
| --- | --- | --- |
| `idle` | 灰色暗灯 | 空闲 |
| `working` | 蓝灯 | 正在执行任务 |
| `done` | 绿灯 | 已完成，等待验收 |
| `waiting` | 红灯 | 等待用户回复、确认、授权 |
| `error` | 红灯 | 执行失败或 hook 异常 |

状态规则：

- `done` 默认 10 分钟后自动回到 `idle`
- `waiting` 前 10 秒红灯脉冲提醒，之后红灯常亮
- `error` 红灯常亮
- 静音、自动回空闲时间、脉冲提醒时间保存在偏好文件中

## 4. 数据文件

状态文件保存在用户数据目录。

macOS：

```text
~/Library/Application Support/CodeAgentTrafficLight/state.json
~/Library/Application Support/CodeAgentTrafficLight/preferences.json
~/Library/Application Support/CodeAgentTrafficLight/command.json
```

Windows：

```text
%APPDATA%\CodeAgentTrafficLight\state.json
%APPDATA%\CodeAgentTrafficLight\preferences.json
%APPDATA%\CodeAgentTrafficLight\command.json
```

也可以通过 `AGENT_LIGHT_HOME` 覆盖数据目录，方便测试。

状态文件结构：

```json
{
  "version": 1,
  "agents": {
    "codex": {
      "state": "working",
      "updatedAt": 1781234567890,
      "message": "PreToolUse"
    },
    "claude": {
      "state": "waiting",
      "updatedAt": 1781234567999,
      "message": "Notification"
    }
  }
}
```

写入策略：

- 所有写入都通过 shared store
- 使用临时文件加 rename 的原子写入
- 非法 agent / state 会被拒绝
- Electron、CLI、hooks 共用同一套读写逻辑

## 5. 进程检测和列展示

列展示完全跟随运行中的 CLI 进程。

检测规则：

- macOS / Linux 使用 `ps -axo pid=,comm=,args=`
- Windows 使用 `Get-CimInstance Win32_Process`
- 只识别明确支持的进程名：`codex`、`claude`
- 排除本应用自身、Electron、hook 脚本、`dist/cli` 等进程

展示规则：

- 0 个 agent 进程：显示一列“未连接”，状态灯为灰色暗灯
- 1 个 agent 进程：显示对应列
- 2 个 agent 进程：显示 `Codex` 和 `Claude`

状态文件不会决定列是否展示。hook 写入过状态但进程不存在时，该 agent 列也不会显示。

## 6. 原型和 UI 设计

原型文件：

```text
prototype-transparent-widget.html
```

原型定位为桌面上的轻量透明玻璃小组件，而不是传统大窗口。界面需要在浅色和深色桌面背景上都保持可辨识，同时尽量减少对用户正在工作的窗口的遮挡。

视觉结构：

- 外层是透明无边框圆角玻璃容器，使用半透明背景、模糊和轻量内阴影建立边界
- 内层按 agent 数量渲染一列或两列，每列包含状态灯、agent 名称和状态文案
- 窗口控制按钮位于右侧，默认隐藏，鼠标悬停或键盘聚焦时出现
- 单 agent 形态用于只有一个 CLI 进程或未连接状态；双 agent 形态用于 Codex 和 Claude 同时运行

尺寸约束：

| 形态 | 宽度 | 高度 | 使用场景 |
| --- | --- | --- | --- |
| 单列 | 202px | 68px | 仅 Codex、仅 Claude、未连接 |
| 双列 | 292px | 68px | Codex 和 Claude 同时运行 |

状态灯视觉：

| 状态 | 视觉表现 |
| --- | --- |
| `idle` | 灰色暗灯 |
| `working` | 蓝色 Autopilot 风格发光，包含呼吸和外圈扩散动效 |
| `done` | 绿色发光 |
| `waiting` | 红色发光，前 10 秒脉冲提醒 |
| `error` | 红色常亮 |

可访问性和交互约束：

- 控制按钮使用真实 `button` 和 `aria-label`
- 动效遵守 `prefers-reduced-motion`
- 外层窗口区域可拖拽，按钮区域设置为不可拖拽
- 文案使用单行省略，避免窄窗口下撑开布局
- “未连接”列只展示状态，不提供状态切换和 agent 菜单

## 7. Hook 状态映射

Hook 入口：

```bash
agent-light-hook <codex|claude> <HookName>
```

实现位置：

```text
src/cli/agent-light-hook.ts
src/shared/hookRules.ts
```

通用映射：

| Hook | 状态 |
| --- | --- |
| `UserPromptSubmit` | `working` |
| `PreToolUse` | `working` |
| `PostToolUse` | `working` |
| `PermissionRequest` | `waiting` |
| `Notification` | `waiting` |
| `Stop` | `done` 或 `waiting` |
| `StopFailure` | `error` |
| `SubagentStart` | `working` |
| `SubagentStop` | `done` |

`Stop` 特殊规则：

- 读取 `last_assistant_message`、`prompt`、`message`、`messages`、`transcript` 等字段
- 如果文本像是在等待用户回复、确认、选择、授权或补充信息，则保持或切到 `waiting`
- Codex 普通等待用户主要依赖 `Stop` 文本载荷判断，也识别 `WaitingOnUserInput`、`InputRequired`、`RequestUserInputQuestionOption` 等内部状态文本
- Claude 的 `Notification` 和 `PermissionRequest` 可以更直接地切到 `waiting`
- 如果当前状态已经是 `waiting`，`Stop` 不会误切到 `done`
- 其他普通 `Stop` 切到 `done`

Codex 和 Claude 的 hook 输出兼容性：

- Codex command hook 成功时 stdout 输出 `{}`，符合 Codex hook output schema
- Claude hook 成功时 stdout 保持为空，避免在 Claude UI 中显示多余输出
- 诊断信息统一输出到 stderr

## 8. Codex 接入方式

Codex 使用 TOML hooks：

```toml
[[hooks.PreToolUse]]
matcher = ".*"
[[hooks.PreToolUse.hooks]]
type = "command"
command = "node \"/Applications/CodeAgentTrafficLight.app/Contents/Resources/app.asar.unpacked/dist/cli/agent-light-hook.js\" codex PreToolUse"
timeout = 5
statusMessage = "CodeAgentTrafficLight: Codex working"
```

配置文件：

```text
~/.codex/config.toml
```

示例：

- `examples.codex-installed-config.toml`：安装后使用
- `examples.codex-dev-config.toml`：本仓库开发调试使用

`notify` 只适合保留给 Computer Use 等通知功能，不用于驱动红绿灯实时状态。

## 9. Claude 接入方式

Claude 使用 JSON hooks：

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "node \"/Applications/CodeAgentTrafficLight.app/Contents/Resources/app.asar.unpacked/dist/cli/agent-light-hook.js\" claude PreToolUse"
          }
        ]
      }
    ]
  }
}
```

配置文件：

```text
~/.claude/settings.json
```

示例：

```text
examples.claude-hooks.json
.claude/settings.local.json
```

## 10. Electron 主进程

主要职责：

- 创建透明置顶悬浮窗
- 创建托盘菜单
- 处理 IPC
- 每秒轮询状态文件、偏好文件、进程检测结果
- 广播状态和 detection 变化给 renderer
- 播放提示音
- 响应 CLI 写入的 quit command

关键文件：

```text
src/main/main.ts
src/main/window.ts
src/main/tray.ts
src/main/preload.ts
src/main/sound.ts
```

窗口可见性策略：

- 创建窗口时 `show: false`
- 等 `ready-to-show` 后 `show()`、`focus()`、`moveTop()`
- macOS 设置 `visibleOnAllWorkspaces`
- 显示后再次设置 `alwaysOnTop`
- 位置计算带屏幕边界保护
- 根据当前检测到的 agent 数量在 202px 和 292px 之间切换宽度
- 宽度变化时尽量保持窗口右边缘位置稳定，并限制在当前屏幕工作区内

窗口属性：

- `frame: false`
- `transparent: true`
- `alwaysOnTop: true`
- `resizable: false`
- `movable: true`

## 11. Renderer

主要职责：

- 渲染 Codex / Claude / 未连接列
- 根据状态点亮单个状态灯
- 处理双击切换状态
- 处理右键菜单请求
- 处理静音、隐藏、关闭按钮
- 监听主进程广播的状态和 detection 变化
- 按原型实现透明玻璃容器、单/双列布局、悬停控制按钮和状态灯动效

关键文件：

```text
src/renderer/App.tsx
src/renderer/display.ts
src/renderer/components/AgentColumn.tsx
src/renderer/components/EmptyAgentColumn.tsx
src/renderer/components/TrafficLight.tsx
src/renderer/styles.css
```

Renderer 展示逻辑：

- `selectVisibleDetections` 只返回当前检测到的 agent
- 如果没有可见 agent，则渲染 `EmptyAgentColumn`
- 根据可见 agent 数量设置 `count-1` 或 `count-2` class
- `AgentColumn` 负责双击循环状态和右键状态菜单
- `TrafficLight` 将状态映射为 `red`、`blue`、`green` 或 `idle`

## 12. CLI

提供三个入口：

```text
agent-light       手动设置状态、查看状态、静音、退出
agent-light-hook  被 Codex / Claude hooks 调用
agent-light-run   包装任意命令并自动切状态
```

关键文件：

```text
src/cli/agent-light.ts
src/cli/agent-light-hook.ts
src/cli/agent-light-run.ts
```

## 14. 打包设计

`package.json` 中的 electron-builder 配置：

- 输出目录：`release/`
- macOS target：`dmg`、`zip`
- `asarUnpack` 包含：
  - `dist/cli/**/*`
  - `dist/shared/**/*`

安装后 hook 脚本路径：

```text
/Applications/CodeAgentTrafficLight.app/Contents/Resources/app.asar.unpacked/dist/cli/agent-light-hook.js
```

这样 Codex / Claude 可以直接通过 `node ".../agent-light-hook.js"` 调用 hook 脚本。
